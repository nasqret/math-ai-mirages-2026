# Production plan

## Acceptance criteria

- [ ] 35–40 slide, 16:9 PowerPoint suitable for a 50-minute technical lecture
- [ ] complete speaker notes and timing map
- [ ] source footer or notes citation for every unstable/public claim
- [ ] no unintended overlap, clipping, empty placeholders, or unreadable text
- [ ] PDF export reviewed slide by slide
- [ ] Jupyter Book builds strictly to HTML
- [ ] Obsidian vault links resolve
- [ ] SageMath, Magma, and Lean examples compile or carry an explicit audited
      environment limitation
- [ ] local validation passes
- [ ] public GitHub repository and live GitHub Pages site verified

## Work packages

1. Inventory supplied repositories, prior slide material, and local reports.
2. Verify current claims from primary sources and freeze a dated evidence ledger.
3. Draft the narrative spine and map every slide to the reference deck.
4. Build the presentation and speaker notes with the artifact-tool workflow.
5. Build the book, vault, executable examples, and landing page.
6. Render, compile, inspect, and repair all deliverables.
7. Publish, enable Pages, and verify the public URL.
